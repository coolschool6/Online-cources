Module 1 Workbook: Foundations & First Prompts

This workbook contains hands-on labs, real-world scenarios, prompt templates, and mini projects for Module 1. Work through them sequentially or jump to what’s most relevant. Each lab includes sample prompts, variations, and optional stretch goals.

Lab 1: Company Bio Generator

Scenario: You need a one-paragraph bio for a startup’s About page.

Prompt:

Write a one-paragraph company bio for [Company], a [industry] startup based in [City].
Audience: prospective customers.
Tone: friendly and trustworthy.
Constraints:
- 120–150 words
- Include founding year and 1–2 standout achievements
- Avoid jargon and buzzwords
Facts:
- Founded in 2022 by engineers from [School]
- Serves 1,200+ SMBs in the US
- Recently won the [Award Name] award

Try variations:

  • Change tone: professional, playful, bold
  • Add a constraint: “readability at Grade 7”
  • Ask for 3 versions and choose the best

Lab 2: Persona Interview

Interview an AI pretending to be your ideal customer to learn their pains.

You are a 35-year-old operations manager at a 50-person e-commerce brand.
Goal: understand your biggest daily frustrations and what would help.
I will ask questions; answer candidly with concrete examples from your day.
Stay in character and mention numbers when relevant.

Ask follow-ups like: “Tell me about the last time that happened—what was the impact?”

Lab 3: Compare-and-Choose

Provide 5 tagline options and have the model select the best with reasoning.

Here are 5 taglines for our project management app for remote teams:
1) Work together, anywhere
2) Less chaos. More clarity.
3) Projects done, promises kept
4) Plan boldly. Deliver quickly.
5) Your team's command center
Evaluate each on clarity, memorability, and differentiation (1–10).
Choose a winner and explain in 3 bullets.

Lab 4: Constraint Practice

Practice strict constraints to see how output quality changes.

Write a product description for eco-friendly laundry detergent.
Constraints:
- 90–110 words
- 2 short paragraphs
- Grade 6 reading level
- Include 3 benefits (bulleted)
- Avoid these words: eco, green, sustainable

Now loosen constraints and compare results. Which is better and why?

Lab 5: Role-Play a Specialist

Use a role to change depth and language.

You are a senior reliability engineer.
Explain the concept of "retry with exponential backoff" to a junior developer.
Include: definition, when to use it, pseudocode, pitfalls, and a 10-line JS code example.

Lab 6: From Vague to Precise

Start with a vague prompt and iteratively refine.

Vague: "Write about remote work challenges."
Refined v1: Audience: startup CTOs; Length: 400 words; Include 3 data points.
Refined v2: Structure with H2s; include 2 quotes; end with a checklist.
Refined v3: Add counterargument section and action-oriented CTA.

Observe how specificity changes relevance and depth.

Lab 7: Structured Output

Enforce a JSON schema.

Summarize this customer email into JSON:
Schema: {
  "intent": string ("complaint"|"question"|"feature_request"),
  "summary": string,
  "urgency": number (1-5),
  "nextSteps": string[]
}
Email: [paste]

Add: “Return only JSON. No prose.” Test strictness.

Lab 8: Evidence & Sources

Ask the model to separate facts from opinions and cite sources when possible.

Topic: "Will AI replace software engineers?"
Write 5 bullet points labeled [Fact] or [Opinion].
If Fact, attach a credible source link.
Add a one-sentence takeaway for a CTO.

Lab 9: Multi-Format Output

Generate both markdown and HTML versions.

Write a "Quick Start" for connecting to our API.
Output two blocks:
1) Markdown version (with fenced code)
2) HTML version (semantic tags only)
Topic details: [auth, base URL, one sample call]

Lab 10: Prompt Anti-Patterns

See what goes wrong with weak prompts.

  • No audience, no constraints
  • Multiple goals in one sentence
  • Ambiguous verbs ("improve", "make better")

Rewrite them into strong, testable prompts.

Mini Project A: Team Handbook Section

Create a 2-page handbook section on "Async Communication for Remote Teams".

  • Audience: engineers and PMs
  • Sections: Principles, Tools, Examples, Do/Don’t
  • Include checklists and 3 templates
Sample Outline

Principles: default to written; clear owners; time-bound asks...

Templates: status update, decision log, RFC comment...

Mini Project B: Meeting Notes Summarizer

Convert raw meeting transcript into structured notes.

Task: Summarize the transcript below.
Output JSON: {
  "title": string,
  "attendees": string[],
  "decisions": string[],
  "actionItems": [{"owner": string, "due": string, "task": string}],
  "openQuestions": string[]
}
Transcript: [paste]

Add tests: “reject if fewer than 2 decisions found; otherwise return schema.”
