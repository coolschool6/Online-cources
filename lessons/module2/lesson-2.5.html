Beyond the prompt itself, you can control how the AI generates responses using parameters like temperature, top-p, and max tokens. This lesson reveals the technical knobs you can turn to fine-tune output creativity, consistency, and length.

Understanding Model Parameters

Think of parameters as settings on a camera: they don't change what you're photographing, but they dramatically affect how the image looks. Similarly, parameters don't change the prompt's meaning but control how the AI interprets and responds to it.

Important Note:

Most chat interfaces (like ChatGPT web) use default settings optimized for general use. Advanced parameters are typically available in API access, AI playgrounds, or platform-specific settings. Not all platforms expose all parameters.

The Big Three Parameters

1. Temperature (Range: 0.0 to 2.0)

What it controls: Randomness and creativity in token selection

Temperature affects how the model chooses the next word (token). Lower values make the model more conservative and deterministic. Higher values increase randomness and creativity.

Temperature Effect on Output Best Use Cases

0.8 - 1.0 More creative, varied, sometimes unexpected Brainstorming, creative writing, marketing copy, fiction
1.1 - 2.0 Highly unpredictable, experimental, potentially incoherent Avant-garde creative projects, random idea generation (rarely used)

2. Top-P / Nucleus Sampling (Range: 0.0 to 1.0)

What it controls: Token selection diversity via probability mass

Top-P (also called "nucleus sampling") determines which subset of possible tokens are considered at each step based on cumulative probability. It's an alternative way to control randomness versus temperature.

  • Top-P 0.1: Only tokens in top 10% probability mass considered (very focused, limited vocabulary)
  • Top-P 0.5: Top 50% probability tokens considered (moderately focused)
  • Top-P 0.9-1.0: Almost all tokens considered up to 90-100% cumulative probability (default for most systems, maximum diversity)

Pro Tip:

Use Temperature OR Top-P, not both aggressively. They serve similar purposes. Most practitioners adjust temperature and leave Top-P at default (0.9-1.0). Adjusting both can produce unpredictable results.

3. Max Tokens

What it controls: Maximum response length

Tokens are the basic units of text (roughly 0.75 words per token on average in English). Setting max tokens prevents runaway responses and controls costs when using paid APIs.

Additional Parameters (Advanced)

Frequency Penalty (Range: 0.0 to 2.0)

Reduces repetition of words/phrases already used in the response. Higher values actively discourage reusing tokens.

  • 0.0: No penalty (model may repeat naturally)
  • 0.5-1.0: Moderate variety, avoids obvious repetition
  • 1.5-2.0: Actively avoids any repetition, maximum lexical diversity

Presence Penalty (Range: 0.0 to 2.0)

Encourages introducing new topics or concepts. Higher values push the model to explore different ideas rather than dwelling on the same subject.

  • 0.0: No encouragement for new topics
  • 0.5-1.0: Moderate topic diversity
  • 1.5-2.0: Strong push toward introducing new concepts

Experimentation Exercise

  1. Set temperature to 0.2 and generate 3 responses
  2. Set temperature to 0.7 and generate 3 responses

Compare the outputs. Notice how temperature 0.2 produces nearly identical results each time, while 1.0 gives wildly different (and possibly less coherent) options.
