Module 2 Workbook: Effective Prompting Patterns

Practice the patterns from Module 2 with realistic, ready-to-run prompts. Each lab includes progressive variations and a mini-evaluation checklist.

Lab 1: Role & Persona

Switch roles and note how outputs change.

Role: Senior Technical Writer.
Task: explain "idempotency" to a junior developer in 150 words.
Tone: clear, friendly, concrete.
Include: 1 analogy, 1 example in HTTP APIs, and a "gotcha" tip.

Try roles: Product Manager, SRE, Support Agent.

Lab 2: Constraints & Requirements

Harden outputs with strict requirements.

Write a changelog entry for version 3.2.0.
Constraints:
- 3 sections: Added, Changed, Fixed
- Each bullet starts with an emoji
- 6–10 bullets total
- Link 2 tickets in [ABC-123] format
- Grade 8 reading level

Validate each constraint was met.

Lab 3: High-Quality Context

Provide real context and see improvement.

Context:
- App: habit tracking mobile app for students
- Problem: low day-1 retention (22%)
- Hypothesis: onboarding is too long
Task: propose 5 onboarding experiments.
For each: rationale, expected impact, measurement metric.

Lab 4: Iterative Refinement

Use a loop: draft → critique → revise.

Write a 200-word feature announcement email for dark mode.
Then critique your own draft against: clarity, benefits, CTA strength, and length.
Finally, produce a revised version addressing all issues.
Return both versions with a short diff summary.

Lab 5: Parameters (Temperature, Top-p)

Compare creativity vs consistency.

Task: 5 subject lines for a webinar invite (AI in Finance).
Run once with temperature=0.2; once with temperature=0.9.
Compare: diversity, quality, and off-topic rate.
Pick the best 2 and explain why.

Lab 6: Schema-First Outputs

Lock down JSON outputs tightly.

Extract product info from the text below.
Return only JSON with schema:
{
  "name": string,
  "price": number,
  "categories": string[],
  "features": string[]
}
Text: [paste 2–3 product blurbs]

Add: "If missing info, set null or []. No extra fields."

Lab 7: Prompt Library

Capture your best prompts.

Create a prompt library entry with fields:
- Name, Goal, Audience
- Systems/Rules
- User Prompt
- Parameters (temp/top_p/max_tokens)
- Examples (few-shot)
- Output spec/schema
- Notes/Learnings

Mini Project: Cold Email Engine

Generate 10 tailored cold emails given a prospect profile.

Input: prospect data (name, company, role, recent news, problems)
Task: create 10 email drafts.
Structure each: subject, opener (custom), value prop, proof, CTA.
Constraints: 110–140 words; no more than 1 emoji; 1-link max.
Return JSON array of objects with those fields.
