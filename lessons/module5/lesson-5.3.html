Time to bring it all together. In this capstone, you’ll scope, design, and deliver a working AI assistant or workflow that solves a real problem. You’ll apply techniques from every module and ship a small but complete system.

Project Brief

Goal:

Build a focused AI solution that delivers value end-to-end in a constrained scope (1–2 weeks of part-time work).

Examples:

  • Marketing: Generate on-brand landing page copy + social snippets from a product brief
  • Support: Turn tickets into structured summaries + suggested responses
  • Docs: Create developer-ready API docs from endpoint specs
  • Education: Tutor that explains code and creates practice questions

Deliverables

  • System prompt (role, purpose, behaviors, format rules)
  • Primary user prompt(s) with examples (few-shot if needed)
  • Structured output schema (JSON, table, etc.)
  • Demo input(s) and outputs (screenshots or text)
  • Safety plan: risks, guardrails, and refusal policy
  • Monitoring plan: metrics, logging fields, and eval set outline
  • Short README (what it does, how to use, limitations)

Rubric (100 pts)

Category Criteria Points
Prompt Quality Clarity, context, constraints, examples 25
Structured Output Valid, consistent schema; parseable 15
Safety Risks identified; guardrails implemented 15
Evaluation Golden set + validators; pass criteria 20
Documentation README clarity; usage steps; limitations 15
Polish Usability; reasonable defaults; examples 10

Templates

System Prompt Template

You are a [role] helping with [goal].
Users are [audience] facing [pain points].
Always follow these rules: [safety, privacy, format].
Respond in [format], max [length].
Ask [1-3] clarifying questions when needed.
If info is missing, explain assumptions.

User Prompt Template

Task: [what to produce]
Context: [facts, constraints]
Style: [tone, voice]
Output: [schema or sections]
Examples: [few-shot pairs]

JSON Schema Example

{
  "title": string,
  "sections": [
    {"heading": string, "content": string}
  ],
  "cta": {"label": string, "url": string}
}

Submission Checklist

  • All deliverables included (prompts, schema, demos, safety, monitoring, README)
  • One-page overview (problem → solution → demo → results → next steps)
  • Clearly state model(s) and parameters
  • Note known limitations and failure cases

Mini Exercise: Scope Your Project

Pick a problem and outline a lean, testable solution.

  1. Write a one-sentence problem statement
  2. List 3 success criteria (what good looks like)
  3. Draft your system prompt and output schema
  4. Define 5 golden examples for evaluation

Problem: Support agents take too long to summarize tickets.

Success: Valid JSON summaries in < 5s with ≥ 90% agent approval.

System prompt: "You are a support summarizer... (rules, schema)"

Golden 5: Mix of short/long, noisy/clean tickets; include edge cases.

Key Takeaway:

Ship something small, real, and well-instrumented. Clear prompts + structured outputs + safety + monitoring = a credible MVP you can iterate on.
