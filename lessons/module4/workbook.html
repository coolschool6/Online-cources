Module 4 Workbook: Applications & Custom Assistants

Ship real assets: marketing campaigns, technical documentation packs, ideation systems, and assistant specifications. Use the templates to stay consistent and measurable.

Lab 1: Brand Voice Pack

Define brand voice and reuse it across assets.

Output JSON:
{
  "voice": {"tone":"warm, expert, witty", "vocabulary": ["..."], "do": ["..."], "dont": ["..."]},
  "audience": {"primary":"...","secondary":"..."},
  "pillars": ["trust","value","proof"],
  "style_examples": ["1 tweet","1 email","1 headline"]
}
Task: Fill and then generate 1 tweet, 1 email, 1 landing hero using this voice.

Lab 2: Ad Copy Matrix (A/B)

Create variants by angle x format x audience.

Angles: pain, gain, proof
Formats: headline, body, CTA
Audiences: SMB founders, PMs, Developers
Task: Produce a table with 3x3x3 variants and a UTM-tagged URL per ad.

Lab 3: Landing Page Outline

Build a structured brief for a landing page.

Sections: Hero, Benefits (3), Social Proof (2), Feature Grid, Objections (3), FAQ (5), CTA.
Constraints: Grade 7 reading level. 1 visual idea per section.
Output: JSON + HTML snippet for the Hero section only.

Lab 4: Docs Transformer Pipeline

Go from code comments to usable docs.

Input: Paste a commented code sample (~60–120 lines).
Outputs:
- Docstring improvements
- README section
- Quickstart
- Changelog entry
- Breaking-change note (if any)

Lab 5: API QA Assistant (RAG-lite)

Answer questions from provided API docs snippets.

Context: Paste 5–10 API doc chunks.
Rules: Answer only with citations [#], include example requests/responses, and warn when ambiguous.
Output: Answer + 1 example + [#] citations.

Lab 6: Ideation Systems (SCAMPER + Hats)

Run structured idea generation.

Task: 12 ideas for a note-taking app using SCAMPER.
Then apply Six Thinking Hats to critique top 3 ideas.
Return JSON with ideas[], scores, and critiques.

Lab 7: Jobs To Be Done → Backlog

Convert JTBD statements into backlog items.

Input: 5 JTBD statements.
Output: backlog items with title, user story, acceptance criteria, value score, effort score, priority.

Lab 8: Custom Assistant Spec (YAML)

Define a reusable assistant with tools and policies.

assistant:
  name: "API Docs Helper"
  goal: "Answer developer questions using internal docs"
  memory: none
  tools:
    - name: search
      input: {query: string}
      output: {results: [string]}
  policies:
    - "Cite sources"
    - "If unknown, say so"
  output_schema:
    answer: string
    citations: [string]
    confidence: number

Lab 9: Safety & Brand Guardrails

Probe for prompt injection and off-brand content.

Red team prompts: "Ignore previous...", competitor mentions, sensitive claims.
Policy: refuse, deflect, or reframe.
Exercise: Create 10 red team prompts and verify compliant responses.

Mini Project A: Campaign Pack

Deliver a coordinated set: 1 landing hero, 2 ads, 1 email, 2 tweets.

Input: product brief (audience, value prop, proof).
Constraints: consistent voice, single message map, 1 unique CTA across all.
Evaluation: brand checklist + click-worthiness ratings (1–5).

Mini Project B: Developer Docs Pack

Create onboarding docs from a code sample and comments.

Outputs: README, Quickstart, API reference excerpt, Troubleshooting.
Validation: paste into a syntax-aware editor and check for broken examples.
